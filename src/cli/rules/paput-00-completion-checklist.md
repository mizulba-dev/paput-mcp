## PaPut Completion Checklist

Run this before writing the final message of any turn in which work was done — a task finished, a problem solved, a design decision settled, memos or notes created, or a repeatable procedure established. Run BOTH checks below, in order, every time. They are independent: one finding nothing says nothing about the other, and doing one is never a substitute for the other.

- [ ] Check 1 — Project document. Did this work settle a project-specific design decision (`design_doc`) or complete a repeatable project procedure (`procedure`)? If yes, save it with `paput_add_project_document` and report what you saved. If no, say "Check 1: no project document needed" so the check is visibly done.
- [ ] Check 2 — Reusable knowledge. Did reusable cross-project knowledge appear? If yes, follow the `paput-capture` workflow and report the title, categories, and candidate ID. If no, say "Check 2: no reusable knowledge found" so the check is visibly done.

State the one-line outcome of EACH check. Never leave a check silent. The detailed procedure for each check is in the sections below, and the self-check at the end of this block confirms both were run.
